## 第1章 绪论（建议 3000~4000 字）

### 1.1 研究背景与意义
- 园区网络规模扩大，设备数量激增，手工安全巡检效率低、易遗漏
- 等保2.0对网络设备安全配置提出明确要求
- 自动化安全评估工具的价值：降低运维成本、标准化安全基线

### 1.2 国内外研究现状
- 国外：Nessus/OpenVAS 偏向漏洞扫描，CIS Benchmark 提供手动检查清单
- 国内：华为/新华三自带 iMC/WSM 网管平台，但部署成本高、学习曲线陡
- 开源领域：Netmiko + Nornir 等自动化框架兴起，但缺少开箱即用的安全评估方案
- **本项目的定位**：轻量级、可定制、面向 H3C 园区网的专项安全评估工具

### 1.3 主要工作
- 设计并实现了分层安全评估模型（通用/路由器/接入/汇聚/核心）
- 基于特征评分的设备角色自动识别算法
- 16 项安全检测规则的 Regex 匹配引擎
- Excel/JSON 双格式可视化报告

### 1.4 论文章节安排
- 简述各章内容

## 第2章 相关技术介绍（建议 2000~3000 字）

### 2.1 Python 语言与生态系统
- Python 在网络自动化领域的优势：丰富的第三方库、简洁的语法
- 本项目使用 Python 3.9+ 的 dataclass、enum、concurrent.futures 等特性

### 2.2 Netmiko 网络设备通信库
- 基于 Paramiko 的多厂商 SSH 连接抽象层
- 支持 H3C Comware V7（`hp_comware` 驱动）
- 连接建立、命令下发、异常处理的封装机制

### 2.3 YAML 配置管理
- 人类可读的配置格式，适合作为规则引擎的输入
- 本项目中设备清单、安全规则、运行参数均采用 YAML

### 2.4 openpyxl 与 Excel 报告生成
- 纯 Python 实现的 Excel 读写库
- 支持图表（饼图、柱状图）、条件格式、单元格样式

### 2.5 正则表达式在文本匹配中的应用
- H3C 设备命令行输出的结构化匹配
- `re.IGNORECASE` 和 `re.MULTILINE` 标志的使用

### 2.6 并发编程 — ThreadPoolExecutor
- IO密集型任务（SSH 连接）的并发加速
- 与 as_completed 模式的配合

## 第6章 系统测试（建议 5000~6000 字）*
*可直接参考 `docs/thesis/测试报告.md`*

### 6.1 测试环境
- 12 台 H3C Comware V7 虚拟设备（HCL 模拟器）
- 包含路由器、核心交换机、汇聚交换机、接入交换机全类型

### 6.2 功能测试
- 设备连接测试（11/12 成功，1 台超时）
- 角色识别测试（全12台判定正确率）
- 规则评估测试（各角色评估覆盖率）

### 6.3 性能测试
- 12 台设备并发评估耗时约 25 秒
- 不同 max_workers 参数下的性能对比

### 6.4 测试结果分析
- 发现了日志审计未开启、DHCP Snooping 未配置、路由认证缺失等多类安全问题
- 与手工检查结果对比验证
