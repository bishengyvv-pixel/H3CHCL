# 需求分析

> 对应论文章节：第3章 需求分析

---

## 1. 业务背景

### 1.1 现状痛点

当前园区网络运维中，安全配置检查依赖人工逐台登录设备核对：

```
安全管理员 → SSH登录设备1 → 逐项检查 → 记录结果
           → SSH登录设备2 → 逐项检查 → 记录结果
           → ... (N台设备)
           → 手工汇总报告
```

**核心问题**：
| 问题 | 影响 |
|------|------|
| 效率低下 | 20 台设备全量检查需 2~4 小时 |
| 标准不一致 | 不同管理员检查深度不同，遗漏风险高 |
| 结果难追溯 | 手工记录零散，无法形成持续性基线 |
| 设备角色混用 | 接入层和核心层的安全要求不同，人工易混淆 |
| 依赖经验 | 新人对 H3C 命令不熟，检查质量下降 |

### 1.2 自动化介入点

```
本系统 → 并发SSH连接12台设备 → 采集配置 → 角色识别
       → 按角色分层评估 → 加权计分 → 生成可视化报告
       → 耗时约25秒（人工需要2小时+）
```

## 2. 功能性需求

### 2.1 用例总览

```
┌──────────────────────────────────────────────┐
│               安全管理员                       │
│  ┌─────────────────────────────────────┐     │
│  │ 1. 配置设备清单 (devices.yaml)       │     │
│  │ 2. 调整检测规则 (rules.yaml)         │     │
│  │ 3. 执行安全评估 (python -m src.main) │     │
│  │ 4. 查看评估报告 (Excel/JSON)         │     │
│  │ 5. 按加固建议修复问题                │     │
│  └─────────────────────────────────────┘     │
└──────────────────────────────────────────────┘
```

### 2.2 功能需求明细

#### FR-01：设备连接管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-01-01 | 支持通过 YAML 配置文件批量录入设备 IP、用户名、密码 | P0 |
| FR-01-02 | 支持通过 SSH 协议并发登录多台设备 | P0 |
| FR-01-03 | 连接失败自动重试 3 次，均失败则标记错误并继续下一台 | P0 |
| FR-01-04 | 支持多网卡环境下指定本机出口 IP（source_ip） | P1 |
| FR-01-05 | 并发数量可配置，批次间支持强制休眠（防HCL模拟器过载） | P1 |

#### FR-02：配置信息采集

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-02-01 | 自动下发 `display version` 获取设备型号 | P0 |
| FR-02-02 | 自动下发 `display current-configuration` 获取运行配置 | P0 |
| FR-02-03 | 自动下发 `display local-user` / `display stp` / `display dhcp snooping` / `display acl all` / `display vlan` | P0 |
| FR-02-04 | 单台设备命令执行失败不影响同批次其他设备 | P1 |

#### FR-03：设备角色识别

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-03-01 | 通过 `display version` 输出识别路由器型号（MSR/SR/CR/AR） | P0 |
| FR-03-02 | 对交换机按 L2 特征（access端口、edged-port、dhcp-snooping）和 L3 特征（Vlan-interface、OSPF、BGP、VRRP）进行评分 | P0 |
| FR-03-03 | 输出角色分类：路由器 / 核心层 / 汇聚层 / 接入层 / 未知 | P0 |
| FR-03-04 | 识别结果以日志形式逐台输出，含各项特征统计 | P1 |

#### FR-04：安全规则评估

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-04-01 | 支持按设备角色加载不同的检测规则集 | P0 |
| FR-04-02 | 支持正则表达式匹配设备配置文本 | P0 |
| FR-04-03 | 支持正向匹配（`find`：找到合规特征则通过）和负向匹配（`not_find`：找到风险特征则扣分） | P0 |
| FR-04-04 | 通用规则对所有角色生效，角色专属规则仅对指定角色生效 | P0 |
| FR-04-05 | 规则库以 YAML 文件存储，支持用户自定义扩展 | P1 |

#### FR-05：风险评分与定级

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-05-01 | 采用加权扣分制：总分 100，每项不合格按权重扣分 | P0 |
| FR-05-02 | 输出三级风险评估：高风险（<60或致命项）、中风险（60~84）、低风险（≥85） | P0 |
| FR-05-03 | 弱口令等致命项直接判定高风险 | P0 |
| FR-05-04 | 分数阈值可通过配置文件调整 | P2 |

#### FR-06：评估报告

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-06-01 | 生成 Excel 报告：含仪表盘（KPI卡片+饼图）、角色分析（柱状图）、设备扣分明细 | P0 |
| FR-06-02 | 每项扣分附带加固命令模板（fix_template） | P0 |
| FR-06-03 | 风险等级以颜色区分（红/黄/绿） | P0 |
| FR-06-04 | 生成结构化的 JSON 报告，便于二次开发或系统集成 | P1 |
| FR-06-05 | PDF 报告（预留接口） | P2 |

## 3. 非功能性需求

### 3.1 可靠性

| 编号 | 要求 |
|------|------|
| NFR-01 | 单台设备连接失败或命令执行异常不得导致整个评估流程崩溃 |
| NFR-02 | 所有异常必须捕获并记录到日志，失败设备标记后跳过继续评估 |

### 3.2 性能

| 编号 | 要求 |
|------|------|
| NFR-03 | 12 台以内设备全流程（连接+采集+评估+报告）应在 60 秒内完成 |
| NFR-04 | SSH 连接超时默认 60 秒，命令执行超时可配置 |

### 3.3 安全性

| 编号 | 要求 |
|------|------|
| NFR-05 | 日志输出中不得明文打印设备密码 |
| NFR-06 | 配置文件中的凭据字段应有注释提醒用户控制文件权限 |

### 3.4 可维护性

| 编号 | 要求 |
|------|------|
| NFR-07 | 规则库与代码解耦，新增检测项无需修改 Python 代码 |
| NFR-08 | 各模块遵循单一职责原则，层间通过 dataclass 传递数据 |
| NFR-09 | 所有日志输出使用中文，降低运维人员阅读门槛 |

### 3.5 可扩展性

| 编号 | 要求 |
|------|------|
| NFR-10 | 报告模块支持新增输出格式（如 PDF、CSV），无需改动其他模块 |
| NFR-11 | Netmiko 设备驱动类型可配置，扩展至其他厂商时只需调整配置 |

## 4. 约束条件

- 目标设备型号：H3C Comware V7 系列（交换机 S58/S51/S55 系列，路由器 MSR/SR 系列）
- 通信协议：仅支持 SSH，暂不支持 Telnet
- 认证方式：静态密码认证，暂不支持 SSH Key
- 运行环境：Python 3.9+，Windows / Linux / macOS 均可
- HCL 模拟器环境需限制并发连接数（建议 ≤5），防止 VirtualBox CPU 过载

## 5. 数据字典

### 5.1 设备对象

| 字段 | 类型 | 说明 |
|------|------|------|
| ip | str | 管理 IP |
| username | str | SSH 用户名 |
| password | str | SSH 密码 |
| source_ip | str | 本机出口 IP（可选） |
| hostname | str | 设备主机名（连接后获取） |
| role | enum | ROUTER/CORE/AGGREGATION/ACCESS/UNKNOWN |
| status | enum | PENDING/SUCCESS/ERROR |

### 5.2 规则对象

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | 规则唯一标识 |
| desc | str | 中文风险描述 |
| check_cmd | str | 采集命令键名 |
| regex | str | Python 正则表达式 |
| weight | int | 扣分权重 |
| match_type | str | find / not_find |
| applicable_roles | list | 适用角色列表 |

### 5.3 评估结果对象

| 字段 | 类型 | 说明 |
|------|------|------|
| device_ip | str | 设备 IP |
| hostname | str | 主机名 |
| role | str | 识别角色 |
| score | int | 安全得分 (0-100) |
| risk_level | str | high / medium / low |
| failed_items | list | 未通过的安全项列表 |
